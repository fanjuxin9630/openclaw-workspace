# 🐭 运财鼠分析框架（2026-06-20 确立）

> 所有分析的标准化流程，确保每个结论可追溯、可验证。

## 一、分析类型与对应方法论

### 1.1 财务成本分析
- **方法论**：成本会计法 + 实际API用量归因
- **学科来源**：管理会计（Managerial Accounting），Horngren 等
- **数据源**：
  - DeepSeek 平台账单截图/API
  - OpenClaw 会话日志（每次工具调用记录）
  - 实际充值记录
- **推导过程**：
  1. 总消费 ÷ 总请求数 = 单次请求平均成本
  2. 按工具类型归类请求
  3. 加权估算各项占比
- **置信度规则**：
  - 高：有原始账单截图或API返回数据
  - 中：有日志但需估算缺失部分
  - 低：纯轮次比例估算，无独立数据源

### 1.2 SWOT 战略分析
- **方法论**：Weihrich 的 TOWS 矩阵（1982）
- **学科来源**：战略管理（Strategic Management）
- **数据源**：
  - 市场调研搜索结果
  - 平台实际数据（Opire API、GitHub API）
  - 实际运营经验
- **推导过程**：
  1. 分别列出内外部因素
  2. 交叉组合生成策略
  3. 按可执行性排序
- **置信度规则**：
  - 高：有平台API数据支撑
  - 中：有搜索来源但未验证
  - 低：纯推理无外部数据

### 1.3 市场调研
- **方法论**：一手数据 + 二手数据交叉验证
- **引用标准**：APA 格式
- **置信度规则**：
  - 高：直接API调用获取的实时数据
  - 中：搜索引擎返回的公开信息
  - 低：间接推断或未经验证的来源

## 三、禁用清单

3. ❌ 不标注置信度
